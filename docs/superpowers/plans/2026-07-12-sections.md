# Sections Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a user-curated store of named sections holding text snippets, independent of the clipboard history.

**Architecture:** Storage is a new `sections.json` written only by the TUI (`config/sections.go`), keeping curated data out of the listener's unlocked rewrite race on the history file. The TUI code lives in a self-contained Bubble Tea sub-model (`app/sections.go`); the root `Model` gains one `screen` field and delegates to it, so the existing 744-line `Update()` and its keybinding-toggling machinery are not extended.

**Tech Stack:** Go 1.24, Bubble Tea, Bubbles (`list`, `textinput`), Lipgloss. No new modules — `textinput` ships with the existing `bubbles` dependency.

**Spec:** `docs/superpowers/specs/2026-07-12-sections-design.md`

**Note on tests:** the repo's six existing test files are empty stubs (`func Test(_ *testing.T) {}`). `tests/config/sections_test.go` will be the first real test file. Tests use the stdlib only (no testify in `go.mod`) and `t.TempDir()` for isolation.

- [ ] **Step 4: Implement `config/sections.go`**

```go
package config

import (
	"encoding/json"
	"fmt"
	"os"
	"strings"

	"github.com/savedra1/clipse/utils"
)

/* Storage for user-defined sections: named groups of saved text snippets.

Kept in its own file (sections.json), separate from clipboard_history.json,
because the clipboard listener rewrites the history file in full on every copy
event with no locking. Sections are written only by the TUI, so curated data is
never in that race. Text only -- no images.
*/

type SectionItem struct {
	Value string `json:"value"`
	Added string `json:"added"` // timestamp, doubles as the item ID
}

type Section struct {
	Name    string        `json:"name"` // unique, the section ID
	Created string        `json:"created"`
	Items   []SectionItem `json:"items"`
}

type Sections struct {
	Sections []Section `json:"sections"`
}

func initSectionsFile() error {
	if _, err := os.Stat(ClipseConfig.SectionsFilePath); !os.IsNotExist(err) {
		return err
	}
	return writeSections(Sections{Sections: []Section{}})
}

// readSections returns the sections on disk. A missing or unparseable file
// yields an empty set rather than an error: a corrupt file must not crash the
// TUI, and must not be overwritten until the user makes an explicit change.
func readSections() Sections {
	data := Sections{Sections: []Section{}}

	raw, err := os.ReadFile(ClipseConfig.SectionsFilePath)
	if err != nil {
		if !os.IsNotExist(err) {
			utils.LogERROR(fmt.Sprintf("failed to read sections file: %s", err))
		}
		return data
	}

	if err := json.Unmarshal(raw, &data); err != nil {
		utils.LogERROR(fmt.Sprintf("failed to parse sections file, treating as empty: %s", err))
		return Sections{Sections: []Section{}}
	}

	if data.Sections == nil {
		data.Sections = []Section{}
	}
	return data
}

// writeSections persists atomically: write to a temp file, then rename over the
// target. Unlike the history file, which is disposable, this is the permanent
// hand-curated store -- a crash mid-write must not be able to destroy it.
func writeSections(data Sections) error {
	encoded, err := json.MarshalIndent(data, "", "    ")
	if err != nil {
		return fmt.Errorf("failed to marshal sections: %w", err)
	}

	if !utils.DiskspaceAvailable(len(encoded)) {
		return fmt.Errorf("not enough disk space to write sections file")
	}

	tmp := ClipseConfig.SectionsFilePath + ".tmp"
	if err := os.WriteFile(tmp, encoded, 0644); err != nil {
		return fmt.Errorf("failed writing sections temp file: %w", err)
	}

	if err := os.Rename(tmp, ClipseConfig.SectionsFilePath); err != nil {
		os.Remove(tmp)
		return fmt.Errorf("failed to replace sections file: %w", err)
	}
	return nil
}

func GetSections() []Section {
	return readSections().Sections
}

func GetSectionItems(name string) []SectionItem {
	for _, s := range readSections().Sections {
		if s.Name == name {
			return s.Items
		}
	}
	return []SectionItem{}
}

func SectionExists(name string) bool {
	for _, s := range readSections().Sections {
		if s.Name == name {
			return true
		}
	}
	return false
}

func AddSection(name string) error {
	name = strings.TrimSpace(name)
	if name == "" {
		return fmt.Errorf("section name cannot be empty")
	}

	data := readSections()
	for _, s := range data.Sections {
		if s.Name == name {
			return fmt.Errorf("section %q already exists", name)
		}
	}

	data.Sections = append(data.Sections, Section{
		Name:    name,
		Created: utils.GetTime(),
		Items:   []SectionItem{},
	})
	return writeSections(data)
}

func DeleteSection(name string) error {
	data := readSections()
	kept := []Section{}
	for _, s := range data.Sections {
		if s.Name != name {
			kept = append(kept, s)
		}
	}
	data.Sections = kept
	return writeSections(data)
}

// AddSectionItem stores a snapshot of the value. Items imported from the
// clipboard history keep no reference back to it: the history evicts entries at
// maxHistory, and a linked item would die with it.
func AddSectionItem(sectionName, value string) error {
	data := readSections()

	for i, s := range data.Sections {
		if s.Name != sectionName {
			continue
		}
		data.Sections[i].Items = append([]SectionItem{{
			Value: utils.SanitizeChars(value),
			Added: utils.GetTime(),
		}}, data.Sections[i].Items...) // newest first, matching the history list
		return writeSections(data)
	}

	return fmt.Errorf("section %q does not exist", sectionName)
}

func DeleteSectionItems(sectionName string, timeStamps []string) error {
	toDelete := make(map[string]bool, len(timeStamps))
	for _, ts := range timeStamps {
		toDelete[ts] = true
	}

	data := readSections()
	for i, s := range data.Sections {
		if s.Name != sectionName {
			continue
		}
		kept := []SectionItem{}
		for _, item := range s.Items {
			if !toDelete[item.Added] {
				kept = append(kept, item)
			}
		}
		data.Sections[i].Items = kept
		return writeSections(data)
	}

	return fmt.Errorf("section %q does not exist", sectionName)
}
```

- [ ] **Step 2: Implement `app/sections.go`**

The sub-model tracks which of its five internal states it is in:

```go
type sectionsState int

const (
	stateSectionList sectionsState = iota // browsing sections
	stateItemList                          // browsing one section's items
	stateNameInput                         // typing a new section name
	stateValueInput                        // typing a new item value
	stateImport                            // picking history entries to import
	stateConfirmDelete                     // confirming a section deletion
)
```

Behavior, per the spec:

- **Section list:** `a` add (name prompt, then drops into the new section), `d` delete
  (via confirmation), `c` copy the whole section (items joined by newlines), `enter`
  drill in, `esc` back to history.
- **Item list:** `a` add by typing, `i` import from history, `d` delete, `c` copy and
  stay, `enter` copy and exit the TUI, `esc` back to the section list.
- **Import:** full history, `/` filter, `tab` pinned-only, multi-select; imports are
  snapshots.

All action keys are gated on `!list.SettingFilter()` so that letter keys type into the
filter when it is active, matching the guard the history list already uses.

Copying goes through `display.DisplayServer.CopyText`, so Wayland/X11/macOS all work
via the abstraction already in place.
